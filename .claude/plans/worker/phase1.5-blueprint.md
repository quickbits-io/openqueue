# Blueprint — Phase 1.5: E2E verification infrastructure

> Produced by the architect agent. One amendment by the orchestrator (marked ⚠ LOCAL PORTS) after discovering both 6379 and 5432 are occupied on this machine by unrelated services.

## (a) Design summary

**One in-process-booted, real-TCP e2e suite under `bun test`, plus a compose stack, plus a minimal Next.js consumer example.** The suite lives in a new private workspace `e2e/`, boots the real `startWorkerApp` from `@openqueue/worker` (Bun-only ⇒ `bun test`, not vitest) on an **ephemeral port (`port: 0`)** inside the test process, and drives every `/openqueue/v1` route via `@openqueue/client` over `http://localhost:<port>` — real Bun.serve, real Redis, real Postgres through `postgresAdapter`. Isolation is **unique namespace per worker boot** for Redis and **drop-schema + `drizzle-kit push` per suite run** for Postgres, exercising the exact documented persistence flow (`defineQueueSchema` schema file + drizzle-kit — `site/content/docs/persistence.mdx`).

Alternatives rejected:

- **Spawned child worker process**: `buildControlApp` samples `NODE_ENV` at construction (`packages/workbench/src/api/v1/app.ts:16`), so locked-mode is covered by set-and-restore around one in-process boot. A child buys only readiness-polling, log parsing, and orphan cleanup. In-process still goes over real TCP.
- **Fixed port / free-port probe**: `startWorkerApp` doesn't expose the bound port, so `port: 0` needs a **one-line contract change** (`WorkerApp.port`) — the only published-code edit; rides the uncommitted Phase 1 train.
- **Raw SQL DDL or `drizzle-kit/api` programmatic push**: drifts from `defineQueueSchema` / semi-public API. `drizzle-kit push` against a schema file importing `defineQueueSchema` from `@openqueue/sdk` is literally the documented user flow — verifying it is a feature.
- **`skipIf` like the workbench integration test**: rejected for e2e deliberately. Workbench's test lives in the default `bun run test` pipeline (must pass without Redis → skip correct there). The e2e suite is a separate, explicitly-invoked pipeline (`bun run e2e`) whose purpose IS the real stack; it **fails loudly** via a `bun test` preload preflight.
- **Compose in CI**: CI already uses GitHub Actions service containers (`.github/workflows/ci.yml`); a new `e2e` job with redis+postgres service containers matches the grain. Compose is the local convenience.
- **examples/nextjs with its own worker**: points at `examples/basic`'s worker (extended with one optional `api.token` env line), triggers the existing `example` task by id.

  redis.disconnect();

**`src/harness.ts`**

```ts
import { randomUUID } from 'node:crypto';
import { fileURLToPath } from 'node:url';
import { createClient, type OpenQueueClient } from '@openqueue/client';
import { defineConfig, postgresAdapter } from '@openqueue/sdk';
import { startWorkerApp } from '@openqueue/worker';
import { drizzle } from 'drizzle-orm/postgres-js';
import postgres from 'postgres';
import { DATABASE_URL, REDIS_URL } from './env';
import { queueSchema } from './queue-schema';

const packageRoot = fileURLToPath(new URL('..', import.meta.url));

export interface TestWorker {
  url: string;
  namespace: string;
  token: string | undefined;
  client: OpenQueueClient;
  db: ReturnType<typeof drizzle>;
  close(): Promise<void>;
}

export async function startTestWorker(
  options: { token?: string | false } = {},
): Promise<TestWorker> {
  const namespace = `e2e-${randomUUID()}`;
  const token =
    options.token === false ? undefined : (options.token ?? `tok-${randomUUID()}`);
  const sql = postgres(DATABASE_URL);
  const db = drizzle(sql);
  const app = await startWorkerApp(
    defineConfig({
      namespace,
      dirs: ['./src/worker'],
      redis: { url: REDIS_URL },
      storage: { adapter: postgresAdapter({ db, schema: queueSchema }) },
      api: token === undefined ? undefined : { token },
    }),
    { cwd: packageRoot, port: 0, signals: false },
  );
  const url = `http://localhost:${app.port}`;
  const client = createClient(
    token === undefined ? { host: url } : { host: url, auth: { bearer: token } },
  );
  return {
    url, namespace, token, client, db,
    close: async () => {
      await app.close();
      await sql.end();
    },
  };
}

/** Boot with NODE_ENV=production and no token — the locked-mode worker.
 *  buildControlApp samples NODE_ENV at construction, so restoring it
 *  immediately after boot is safe. If Phase 2 moves auth to per-request env
 *  reads, switch this to a spawned child process. */
export async function startLockedWorker(): Promise<TestWorker> {
  const prev = process.env.NODE_ENV;
  process.env.NODE_ENV = 'production';
  try {
    return await startTestWorker({ token: false });
  } finally {
    if (prev === undefined) delete process.env.NODE_ENV;
    else process.env.NODE_ENV = prev;
  }
}
```

## (e) Risks

- **`WorkerApp.port`** is new public surface on `@openqueue/worker` — tiny, additive, rides the uncommitted Phase 1 train, but forever once published. Only published-code change in this phase.
- **Port collisions locally**: mitigated by parameterized compose host ports (orchestrator amendment); suite honors `REDIS_URL`/`DATABASE_URL` overrides.
- **`drizzle-kit push` loading the sdk import graph**: fallback to `@openqueue/core/drizzle`, but surface it as a doc/product bug (documented flow broken).
- **Global runtime binding is process-wide**: suite depends on `bun test` serial single-process execution; bound-runtime assertions confined to `sdk-binding.test.ts` with unbind in `afterAll`.
- **`NODE_ENV` mutation** safe only while `buildControlApp` samples at construction — comment in harness; Phase 2 must revisit.
- **Raw `bun test` from repo root** picks up e2e and fails loudly without services — acceptable; repo convention is `bun run test`.
- **Turbo `test` env fix** busts the test cache (REDIS_URL now hashes) — intended; kills a false green.
- **Redis residue** under dead `e2e-*` namespaces on a standing stack — bounded by core's runs-cache TTL/caps; `docker compose down -v` is the reset. No `FLUSHDB` anywhere (shared stack safety).
- **CI time**: `e2e` job rebuilds packages; `check` now builds the Next example. Scope later if it hurts — don't pre-optimize.
- **Next 16 specifics** (async `params`, server actions) — read the installed `node_modules/next/dist/docs/`, per `site/AGENTS.md`.

Key reference files: `packages/worker/src/index.ts` (boot/close/validateConfig), `packages/workbench/src/api/v1/{app,auth,routes}.ts` (auth semantics + route statuses), `packages/client/src/{client,http}.ts` (error-code mapping: 401→`unauthorized`, 404→`not_found`, cancel 409→outcome), `site/content/docs/persistence.mdx` (the drizzle-kit flow e2e mirrors), `.github/workflows/ci.yml` (service-container + pinned-SHA conventions).
